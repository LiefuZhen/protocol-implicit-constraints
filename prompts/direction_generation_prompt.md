# 约束生成方向归纳提示词

用途：从已经复核过的隐式 CVE 中归纳“约束生成方向”。

聚类重点是“标准哪里留空子、如何反推约束”，而非漏洞危害或 CWE。

## 输入

- 一组已经 T2 复核过的 CVE 记录；
- 每条 CVE 的标准映射；
- 每条 CVE 的候选隐式约束；
- 人工复核结论。

## 聚类依据

请按以下维度聚类：

1. 标准缺陷形态：沉默、含糊、欠定义、前后矛盾、目的未落地；
2. 在标准中如何定位这种位置；
3. 从该位置如何反推出约束；
4. 绑定的协议目标；
5. 误报风险。

## 每条方向输出

每条方向至少包括：

- `direction_id`
- `direction_name`
- 中文解释；
- 适用的标准缺陷形态；
- 在新 RFC/标准中如何查找；
- 反推隐式约束的规则；
- 支撑它的 CVE seed；
- 适用边界；
- 误报风险；
- 是否建议进入 direction set v0。

## 输出要求

- 方向集完整性标记为阶段性结论；
- CWE 可作为辅助标签；
- 普通内存安全 bug 需先确认标准合规关系；
- 证据不足的方向标记为待人工复核。
